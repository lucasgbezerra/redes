Questões DNS

  1. Qual é o retorno do comando dnsdomainname? O que significa?
    R: O comando dnsdomainname retorna o nome do domínio DNS do sistema. Ele permite que o sistema resolva nomes de host em endereços IP usando o DNS, o que é essencial para a comunicação em rede.

  2. O que é o nome localhost? E o endereço 127.0.0.1 dado a ele no arquivo /etc/hosts? Por que deve sempre existir este endereço e nome em sistemas UNIX/Linux?
    R: No contexto de Redes de Computadores, localhost significa uma referência para o próprio computador configurado no arquivo ‘/etc/hosts’, e o endereço 127.0.0.1 é o IP atribuído ao próprio computador. Essas definições são importantes para que o computador tenha um acesso a rede de loopback, que aponta para o próprio computador em questão. Através dessas definições, é possível checar o status dos serviços de redes do computador; desenvolver localmente com comunicações internas no computador, e também permite que um serviço se comunique dentro de uma máquina, sem precisar ser exposto externo ao computador,

  3. O que é o FQDN?
    Nome de Domínio Totalmente Qualificado (FQDN na sigla em inglês), é um nome de domínio completo, que consiste em um hostname e um nome de domínio que pode ser atribuído a um endereço de IP. O domínio absoluto é divido em partes com hierarquia. Que são elas:

Hostname, que é como uma “etiqueta” que é utilizado para um para que, pelo DNS, torne o endereço do IP mais fácil de ser memorizado. São exemplos de Hostname o “www” e o “br”.
Subdomínio: É a parte localizada mais à esquerda do domínio de segundo-nível, e pode ser uma seção maior que o próprio domínio. Ele não é essencial, ou seja, pode ou não existir, e vem logo após o hostname e antes do domínio em si. Por xempo temos o docs.google.com, o “docs” é o subdomínio.
Nome de domínio. Consiste em domínio de topo e um domínio de segundo nível, como por exemplo docs.google.com, “google” é o domínio de segundo nível e a extensão “.com” indica o domínio de topo.

O comprimento máximo de um hostname e de um nome de domínio totalmente qualificado é de 63 bytes por rótulo e 255 bytes por FQDN. Isto é, o nome não pode ultrapassar 63 caracteres.
4) Podemos ter 2 servidores DNS na mesma rede? Qual é a configuração mais adequada para esta situação?
R: Sim, é possível configurar 2 servidores DNS na mesma rede, classificando eles em primário e secundário. Nesse caso, com 2 servidores DNS na mesma rede, ao tentar acessar um determinado endereço, será utilizado o servidor de DNS primário, e em caso de erro ou manutenção no primário, será utilizado o servidor de DNS secundário para resolver o nome a ser acessado. Isso ajuda a criar uma redundância dentro da rede para resolver o DNS dos IPs a serem acessados.

  1. O que é a entrada MX inserida no domínio? Podem haver mais de uma?
    R: A sigla MX significa Mail Exchange, que representa o registro DNS em um domínio, especificando o servidor de emails responsável por gerenciá-los. Em resumo a entrada MX inserida em um domínio, representa para onde devem ser entregues os emails para um determinado domínio. Sim, pode haver mais de uma entrada MX inserida no domínio, através do uso de prioridades numéricas, ou seja, quando um servidor tentar entregar um email para um domínio com duas entradas, vai tentar a mais prioritária, caso não consiga, tentará na segunda e assim sucessivamente, conforme quantas entradas MX existirem para o domínio em questão.

  2. O que é resposta autoritativa dada por um servidor DNS? Explique.
    Resposta autoritativa é uma resposta dada pelo servidor que tem autoridade sobre o domínio para qual a consulta é feita. Quando o usuário faz uma consulta DNS, o servidor primeiro tenta encontrar uma resposta em seu cache, se não conseguir ele enviará uma consulta para outro servidor DNS, que será o servidor DNS autoritativo, se ele tiver as informações solicitas, ele fornecerá uma resposta autoritativa ao usuário, que contem o endereço IP do host que corresponde ao nome de domínio que o usuário está tentando acessar. Se o DNS autoritativo não tiver as informações ele fornecerá uma resposta negativa que contém um código de erro que indica porque o servidor DNS não pôde fornecer a resposta solicitada. As respostas autoritativas são importantes pois fornecem ao usuário as informações mais precisas e atualizadas sobre os domínios que ela está fazendo consultas.

  3. O que é um servidor caching-only?
    R: Um servidor DNS caching-only, é um servidor que apenas consulta e retorna resultados, mantendo um cache local. Ele não é responsável por nenhum domínio, existe para somente para melhorar as resoluções dos nomes para os clientes locais, através do seu cache.

  1. Qual a diferença entre o armazenamento de e-mails no formato mbox e maildir? Instale o maildir e descreva quais os procedimentos de instalação de um e outro caso.
    MBOX é um formato de arquivo de texto simples que armazena os e-mails em uma única linha. Ele é fácil de ler e escrever e é compatível com a maioria dos clientes de e-mail. No entanto, não é muito eficiente em termos de espaço, pois armazena todos os e-mails em um único arquivo.
    Maildir é um formato de arquivo mais moderno que armazena e-mails em arquivos separados. É mais eficiente em termos de espaço e também mais tolerante a erros. No entanto, não é tão fácil de ler e escrever e não é compatível com todos os clientes de e-mail.
    Para instalar o maildir é simples ele acompanha o dovecot, portanto basta fazer o comando:
sudo apt install dovecot-core dovecot-imapd dovecot-pop3d

  1. Para que servem os esquemas de autenticação SASL/TLS. Caso tenha instalado, descreva qual foi a solução adotada (Dovecot ou outro) e explique os passos de instalação.
    R: Os esquemas de autenticação SASL/TLS são usados para garantir a segurança e autenticidade de conexões em redes, principalmente na comunicação cliente-servidor. A sigla SASL significa “Simple Authentication and Security Layer”, e a sigla TLS significa “Transport Layer Security”. O uso de ambos em uma comunicação fornece um mecanismo robusto para a autenticação e proteção dos dados que percorrem a rede.

  2. O que é a diferença entre os formatos RFC822 e MIME types definido para e-mails? Explicá-los.
    RFC 822: é um protocolo de e-mail que define estrutura de mensagens de e-mail. Ele é o formato mais antigo de e-mail ainda em uso, e foi publicado pela primeira vez em 1982. Ele define mensagens de e-mail tendo um cabeçalho e um corpo.
    Já o MIME é uma extensão do RFC822 que foi publicado pela primeira vez em 1986. Ele adiciona suporte para formato de texto não ASCII, também para formatos de arquivo binário como imagens, áudio e vídeo e para suporte para anexos de arquivos em mensagens de e-mail.
